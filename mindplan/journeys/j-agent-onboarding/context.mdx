# Agent onboarding

Adopt MindPlan in a consumer project and wire it into coding agents via init, playbooks, and integrations.

## Overview

A developer installs or builds the MCP server, runs `mindplan-mcp init` in a project, registers the server with their coding agent, and follows the always-on playbook + define-entities skill. Distribution today is from source; npm publish is the remaining gap.

## Linked Workflows

- `wf-project-init` — scaffold `mindplan/` and agent assets
- `wf-agent-integrations` — Cursor, Claude Code, Copilot, Windsurf, Cline, Continue, generic
- `wf-framework-docs` — SPEC + README quick start
- `wf-npm-publish` — publish `mindplan-mcp` to npm (**not shipped**)
