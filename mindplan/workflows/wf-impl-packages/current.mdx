# Prescribe implementation packages

## Purpose

Make architecture scream from the filesystem: every Foundation and Workflow owns a prescribed package under `src/foundations/<id>/` or `src/workflows/<id>/`. Thin `src/index.ts` boots `f-mcp-runtime`. Suffix (`-v2`/`-v3`) Workflows are deprecated; orientation excludes retired nodes.

## Execution Logic

This Workflow is process + layout (package pointer under `src/workflows/wf-impl-packages/`). Live contract:

1. Foundations live under `src/foundations/<id>/` (domain, compiler-rules, territory-store, graph-search, view-projection, mcp-runtime)
2. Workflows live under `src/workflows/<id>/` (query, mutate, export, project-init, test-harness, …)
3. `create_node` / store scaffolds those packages; `get_node_implementation` returns the root
4. `npm test` points at `wf-test-harness`
5. Deprecated suffix Workflows remain in the graph for history but are not orientation focus

Depends on: `f-territory-store`, `f-mcp-runtime`. Belongs to: `j-territory-sdlc`.

## Acceptance Criteria

- [x] All runtime Foundations/Workflows have prescribed packages matching their ids
- [x] Thin entry boots MCP runtime; imports resolve from packages
- [x] `get_node_implementation` reports package roots
- [x] Stale suffix Workflows are `deprecated`; map/orientation prefer clean ids
- [x] `npm test` green
