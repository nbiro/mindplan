# Query graph

Read orientation tools: find_related_nodes, get_mindplan_graph, get_node_context, and get_blast_radius — with blast radius aware of version lineage.

## Execution Logic

1. Agent calls `find_related_nodes` with a query (preferred orientation).
2. Optionally `get_node_context` for full territory body/attachments.
3. `get_blast_radius` for transitive dependents before risky changes (same stable id; no supersedes seeding).
4. `get_mindplan_graph` only for greenfield/full audits.

Ranking lives in `src/search.ts`; blast radius and assembly via `src/rules.ts` / store `loadGraph`.
