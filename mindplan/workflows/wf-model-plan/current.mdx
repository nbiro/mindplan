# Model the plan

## Purpose

Model the plan — human UI for define-entities / plan-project: create Journeys, Foundations, Workflows, and Bugs; set title/description; establish `belongs_to` / `depends_on` / `affects`. Journey-first: refuse Workflow creation when no matching Journey exists.

## Execution Logic

Package: `src/workflows/wf-model-plan/`

- Create forms call bridge `create_node` with type + id + title + description
- Link helpers enforce legal edge shapes; Foundation role guidance in copy (Assembler / Infra / Design system / Adapter)
- After create/link, refresh graph explore so the new structure is visible
- Enrichment of territory body may deep-link to territory viewer; prose edits can remain host-file-tools in v1 if bridge patch is deferred — document the chosen path in implementation
- Stays at modeling intent: does not own lifecycle “ship” review theatre (that is `wf-console-mutate`)

Belongs to: `j-plan-console`. Depends on: `wf-console-shell`, `wf-graph-explore`, `f-console-bridge`, `f-design-system`.

## Acceptance Criteria

- User can create a Journey then a Workflow linked with `belongs_to`
- Workflow create is blocked in the UI when no Journey exists (mirror playbook)
- New nodes appear on the XYFlow graph after refresh
- Compiler rejects illegal links with visible `Blocked:`
