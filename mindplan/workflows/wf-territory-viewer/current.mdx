# Read territory

## Purpose

See the plan — read a node’s PRD / shared substrate spec, acceptance criteria, Atomic Ops, and attachments. Implements SPEC §6.4 reserved MDX components for presentation; Markdown `- [ ]` / `- [x]` remain the Completion Check authority.

## Execution Logic

Package: `src/workflows/wf-territory-viewer/`

- Load `get_node_context`-equivalent payload via `f-console-bridge` (record + body + optional `next`)
- Render body with reserved components: `StateBadge`, `AtomicOp`, `AcceptanceCriteria`, `Attachment`, edge refs, Bug helpers
- Show live vs `next` slot when evolving
- Read-only in this Workflow — edits belong to mutate / model (or host file tools outside the console)

Belongs to: `j-plan-console`. Depends on: `wf-console-shell`, `f-console-bridge`, `f-design-system`.

## Acceptance Criteria

- Selecting a node shows title, state, description, and full body
- Reserved MDX components render; unknown project components degrade gracefully
- Open `next` slot is visible when present
- Checkbox display matches territory Markdown (no JSX `done` override of compiler gates)

- [ ] Reserved MDX component renderers per SPEC §6.4
