# See the map

## Purpose

Human-facing graph diagrams: on-demand Mermaid/DOT export and automatic persistence of `mindplan/map.md` after successful graph mutations so the repo always has a readable map (MCP remains authority).

## Execution Logic

Package: `src/workflows/wf-export-views/`

- `tools.ts` — ownership list `EXPORT_TOOLS`: `export_mindplan_view`
- Projection substrate: `f-view-projection` (`graphToMermaid` / `graphToDot` / `persistMindPlanMap`)
- Registration and post-mutation map refresh: `f-mcp-runtime/server.ts`

Depends on: `f-mcp-runtime`, `f-territory-store`, `f-view-projection`. Belongs to: `j-territory-sdlc`.

## Acceptance Criteria

- [x] `export_mindplan_view` returns Mermaid and/or DOT (full or focused)
- [x] Graph mutations that succeed update `mindplan/map.md`
- [x] Agents treat MCP export / orientation as authority, not the map file alone
- [x] Verified via `npm test` (`view.test.mjs`)
