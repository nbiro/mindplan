# Framework documentation

## Purpose

Own MindPlan’s normative documentation surfaces so agents and humans share one process: taxonomy, compiler gates, Territory Completeness, Foundation roles, and how to evolve shipped work without destroying the living contract.

## Execution Logic

Docs live at repo root and under `templates/agent/` / `mindplan/agent/`. The package folder `src/workflows/wf-framework-docs/` is a pointer (no runtime TS).

Surfaces owned by this Workflow:

- `SPEC.md` — normative language (taxonomy, versioning, Territory Completeness, compiler rules)
- `AGENTS.md` — always-applied agent playbook (repo root)
- `templates/agent/playbook.md` — consumer playbook shipped by `mindplan-mcp init`
- `mindplan/agent/playbook.md` — local mirror of the playbook
- `templates/agent/skills/define-entities/` (+ mindplan/agent mirror) — entity scaffolding skill
- `README.md` — product framing and taxonomy summary

Foundation scaffold copy that mentions roles lives in `src/foundations/f-territory-store/store.ts` (touched when role/description conventions change).

### Territory Completeness

`current.mdx` is the complete contract of what a node *is* in the repository now. `next.mdx` is a complete proposed successor of that same contract (seeded from current, edited in place). Atomic Ops on `next` may be evolution-scoped; PRD/spec sections must remain a full post-ship document. Delta-only or changelog-only `next` bodies are forbidden because `ship` promotes `next` over `current`.

### Foundation roles (docs convention)

Assembler, Infra, Design system, Adapter — documentation groupings only (not NodeTypes). Agents SHOULD tag the role at the start of a Foundation `description`.

## Acceptance Criteria

- SPEC documents Territory Completeness and Foundation roles.
- Playbooks (AGENTS + templates + mindplan mirror) instruct agents to keep `current`/`next` as full contracts and forbid delta-only promotion.
- define-entities skill aligns with Foundation role tagging.
- README summarizes Foundation roles in the taxonomy section.
- Active MindPlan Journey/Foundation/Workflow territories describe full repo state (not stubs or latest-change deltas).
