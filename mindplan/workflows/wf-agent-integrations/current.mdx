# Agent integrations

## Purpose

Per-agent MCP setup guides so a consumer can wire MindPlan into Cursor, Claude Code, Codex, Copilot, Windsurf, Cline, Continue, or a generic MCP host — plus an index README covering the territory authority split and how to review MCP vs file-tool edits.

## Execution Logic

- Templates: `templates/agent/integrations/` (`README.md`, `cursor.md`, `claude-code.md`, `codex.md`, `copilot-vscode.md`, `windsurf.md`, `cline.md`, `continue.md`, `generic.md`)
- Installed into `mindplan/agent/integrations/` by `wf-project-init` (`installAgentIntegrations`)
- Package folder `src/workflows/wf-agent-integrations/` is a pointer (docs-only Workflow)

Each guide covers MCP registration for that agent and where playbook/skill assets land after init. Cursor (and the shared README) document: MCP for graph mutations; host file tools for territory prose; `.cursorignore` still ignores `mindplan/map.md` (and `mindplan/agent/**`) but not `current.mdx`/`next.mdx`; graph tool results include `changed_files`; review prose via native edit UI / Source Control.

Depends on: `f-mcp-runtime`. Belongs to: `j-agent-onboarding`.

## Acceptance Criteria

- Guides exist for Cursor, Claude Code, Codex, Copilot, Windsurf, Cline, Continue, and generic MCP
- Integrations README indexes the set and notes authority split / `changed_files`
- Cursor guide documents ignore policy, authority split, and review surfaces
- Init installs the guides into the consumer `mindplan/agent/integrations/` tree
- Covered by init path in smoke tests
