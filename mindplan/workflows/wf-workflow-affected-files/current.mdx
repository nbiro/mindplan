# Workflow affected files

Workflows maintain a territory-body list of project files touched during implementation. Agents query per-workflow file lists through MCP `get_workflow_files`.

## Execution Logic

1. Workflow scaffold includes an `## Affected Files` section (empty list).
2. During build-pipeline **Execute**, agents append project-relative paths to that section as they modify application code.
3. `parseAffectedFiles` in the territory store reads the section from `current.mdx` (markdown list items until the next `##` heading).
4. MCP tool `get_workflow_files({ node_id })` returns `{ node_id, files: [...] }` for Workflow nodes only.
5. Playbook and SPEC document the convention and tool.
