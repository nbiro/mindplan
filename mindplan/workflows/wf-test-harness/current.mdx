# Test harness

## Purpose

Regression suite for MindPlan: unit tests for view/search and an MCP stdio smoke path that also exercises `mindplan-mcp init` template install. `npm test` builds then runs this harness. Covers `force_unship` recovery gates, init install of `.cursor/permissions.json`, and Cursor-native skill/rule paths under `.cursor/skills/` and `.cursor/rules/mindplan.mdc`.

## Execution Logic

Package: `src/workflows/wf-test-harness/`

- `view.test.mjs` — Mermaid/DOT projection unit tests
- `search.test.mjs` — ranking excludes retired nodes
- `check.test.mjs` — integrity check CLI (orphan, dirty-src, `--for-main`)
- `smoke.mjs` — MCP stdio end-to-end + init + `force_unship` + `cancelled` gates; after CLI `init` in a temp root, assert:
  - existing: playbook, AGENTS.md, `.cursorignore`, `.cursor/permissions.json`, integrations, `mindplan/agent/skills/{plan-project,review-work}/SKILL.md`, default layout config
  - new: `.cursor/skills/mindplan-define-entities/SKILL.md`, `.cursor/skills/mindplan-plan-project/SKILL.md`, `.cursor/skills/mindplan-review-work/SKILL.md`, `.cursor/rules/mindplan.mdc` (contains `alwaysApply: true` and playbook substance)

Depends on: `f-mcp-runtime` (and transitively store/search/view). Belongs to: `j-territory-sdlc`. Paired with `wf-project-init` + `f-mcp-runtime` nexts for `bug-init-cursor-skills`.

## Acceptance Criteria

- `npm test` runs build + view + search + check + smoke successfully
- Smoke covers cancel (dependents block, terminal, view hide/`include_retired`)
- Smoke asserts `mindplan-mcp init` installs `review-work` skill (alongside plan-project) under `mindplan/agent/skills/`
- Smoke asserts `mindplan-mcp init` installs Cursor-native `.cursor/skills/mindplan-*/SKILL.md` and `.cursor/rules/mindplan.mdc`
- Check harness covers orphan package, dirty ownership, `--for-main`
