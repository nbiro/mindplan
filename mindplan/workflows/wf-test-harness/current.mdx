# Test harness

## Purpose

Regression suite for MindPlan: unit tests for view/search and an MCP stdio smoke path that also exercises `mindplan-mcp init` template install. `npm test` builds then runs this harness. Covers init install of `.cursor/permissions.json`.

## Execution Logic

Package: `src/workflows/wf-test-harness/`

- `view.test.mjs` — Mermaid/DOT projection unit tests
- `search.test.mjs` — ranking excludes retired nodes
- `smoke.mjs` — MCP stdio end-to-end + init template install (including `.cursor/permissions.json` with `mindplan:*`)

Depends on: `f-mcp-runtime` (and transitively store/search/view). Belongs to: `j-territory-sdlc`.

## Acceptance Criteria

- `npm test` runs build + view + search + smoke successfully
- Smoke covers MCP tool round-trips and consumer init assets
- Smoke asserts init installs `.cursor/permissions.json` with `mindplan:*`
- Search tests assert retired-node exclusion from orientation ranking
