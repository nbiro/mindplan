# Test harness

## Purpose

Regression suite for MindPlan: unit tests for view/search and an MCP stdio smoke path that also exercises `mindplan-mcp init` template install. `npm test` builds then runs this harness. Covers `force_unship` recovery gates and init install of `.cursor/permissions.json`.

## Execution Logic

Package: `src/workflows/wf-test-harness/`

- `view.test.mjs` — Mermaid/DOT projection unit tests
- `search.test.mjs` — ranking excludes retired nodes
- `check.test.mjs` — integrity check CLI (orphan, dirty-src, `--for-main`)
- `smoke.mjs` — MCP stdio end-to-end + init + `force_unship` + `cancelled` gates

Depends on: `f-mcp-runtime` (and transitively store/search/view). Belongs to: `j-territory-sdlc`.

## Acceptance Criteria

- `npm test` runs build + view + search + check + smoke successfully
- Smoke covers cancel (dependents block, terminal, view hide/`include_retired`)
- Check harness covers orphan package, dirty ownership, `--for-main`
