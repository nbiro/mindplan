# Test harness

## Purpose

Regression suite for MindPlan: unit tests for view/search and an MCP stdio smoke path that also exercises `mindplan-mcp init` template install. `npm test` builds then runs this harness. Covers `force_unship` recovery gates, init install of `.cursor/permissions.json`, Cursor-native skill/rule paths under `.cursor/skills/` and `.cursor/rules/mindplan.mdc`, and `init -f` / `--force` overwriting previously installed agent assets after an upgrade.

## Execution Logic

Package: `src/workflows/wf-test-harness/`

- `view.test.mjs` — Mermaid/DOT projection unit tests
- `search.test.mjs` — ranking excludes retired nodes
- `check.test.mjs` — integrity check CLI (orphan, dirty-src, `--for-main`)
- `smoke.mjs` — MCP stdio end-to-end + init + `force_unship` + `cancelled` gates; after CLI `init` in a temp root, assert:
  - existing: playbook, AGENTS.md, `.cursorignore`, `.cursor/permissions.json`, integrations, `mindplan/agent/skills/{plan-project,review-work,code-review}/SKILL.md`, default layout config
  - Cursor: `.cursor/skills/mindplan-define-entities|plan-project|review-work|code-review/SKILL.md`, `.cursor/rules/mindplan.mdc`
  - Force overwrite: after a successful init, mutate a known installed file (e.g. playbook and/or a Cursor skill), run `init -f` (and/or `--force`), assert the file content is restored from the package template; assert bare `init` (no `-f`) does **not** restore a mutated file; assert `-f` alone does not flip an existing layout-free `config.json`

Depends on: `f-mcp-runtime` (and transitively store/search/view). Belongs to: `j-territory-sdlc`. Paired with `wf-project-init` + `f-mcp-runtime` nexts for init force overwrite.

## Acceptance Criteria

- `npm test` runs build + view + search + check + smoke successfully
- Smoke covers cancel (dependents block, terminal, view hide/`include_retired`)
- Smoke asserts `mindplan-mcp init` installs `review-work` and thin `code-review` under `mindplan/agent/skills/` and `.cursor/skills/mindplan-code-review/`
- Smoke asserts `mindplan-mcp init` installs Cursor-native `.cursor/skills/mindplan-*/SKILL.md` and `.cursor/rules/mindplan.mdc`
- Smoke asserts `mindplan-mcp init -f` overwrites a previously mutated agent asset from templates; bare re-init does not
- Smoke asserts `init -f` preserves existing `implementation_packages: "off"` when `--layout` is not passed
- Check harness covers orphan package, dirty ownership, `--for-main`
