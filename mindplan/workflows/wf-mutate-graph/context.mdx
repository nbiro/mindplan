# Mutate graph

Write-path tools that mutate territory: create_node, create_node_version, link_nodes, unlink_nodes, update_node_status.

## Execution Logic

1. `create_node` scaffolds folder + `context.mdx` at initial state.
2. `link_nodes` / `unlink_nodes` update source frontmatter edge arrays with taxonomy + cycle checks.
3. `update_node_status` applies build/bug transitions; `ship` sets `shipped_at` and computes stability.
4. `create_node_version` supersedes a shipped node, inherits outgoing edges, duplicates incoming `depends_on`.
5. After mutations, journey states and stability are recomputed and synced.
