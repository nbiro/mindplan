# Mutate graph

## Purpose

Write-path MindPlan tools: create/link nodes, advance pipeline state, patch territory, and open/discard/promote evolutions — always through the compiler.

## Execution Logic

Package: `src/workflows/wf-mutate-graph/`

- `tools.ts` — ownership list `MUTATE_TOOLS` registered by `f-mcp-runtime/server.ts`:
  - `create_node`, `link_nodes`, `unlink_nodes`
  - `update_node_status` (current or `next` slot; `ship` promotes `next`)
  - `patch_node_territory` (body, title/description, checkbox toggles; defaults to `next` when evolving)
  - `open_next`, `discard_next`

Compiler gates live in `f-compiler-rules`. Depends on: `f-mcp-runtime`, `f-compiler-rules`. Belongs to: `j-territory-sdlc`.

## Acceptance Criteria

- [x] All seven mutate tools are registered and persist through the territory store
- [x] Illegal mutations return `Blocked:` and leave the graph unchanged
- [x] Successful mutations refresh `mindplan/map.md` via the runtime
- [x] Verified via `npm test`
