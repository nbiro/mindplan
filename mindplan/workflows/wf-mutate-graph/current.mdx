# Mutate graph

Write-path tools that mutate territory: create_node, open_next, discard_next, link_nodes, unlink_nodes, update_node_status, patch_node_territory.

## Execution Logic

1. `create_node` scaffolds folder + `current.mdx` at initial state.
2. `link_nodes` / `unlink_nodes` update source frontmatter edge arrays with taxonomy + cycle checks (writes to `next.mdx` when evolving).
3. `update_node_status` applies build/bug transitions; `ship` sets `shipped_at` and computes stability. When `next.mdx` exists, build transitions apply to next and `ship` promotes next over current.
4. `open_next` opens a draft `next.mdx` on a shipped Foundation/Workflow (same id); `discard_next` abandons it.
5. After mutations, journey states and stability are recomputed and synced.
