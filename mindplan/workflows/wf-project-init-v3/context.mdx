# Project init (external review before ship)

Scaffold consumer mindplan/ plus agent playbook that forbids the implementing agent from shipping: `in-review` waits for a human or another agent to review, then `ship`.

## Execution Logic

1. Run `node dist/index.js init` (or `mindplan-mcp init`) from project root.
2. Creates `mindplan/` type dirs and installs from `templates/agent/`:
   - playbook (with blast-radius orient + external review before ship), define-entities skill, mcp.json.example, integrations guides
3. Creates root `AGENTS.md` only when missing (from the playbook template).

Playbook change: after Atomic Ops are complete, the implementing agent moves the node to `in-review` and **stops**. Only a human or a different agent may `ship` (Foundation/Workflow) or `resolved` (Bug). Documented in Review gate / Ship steps, Bug lifecycle, compiler-rules table (External Review), and Never do.
