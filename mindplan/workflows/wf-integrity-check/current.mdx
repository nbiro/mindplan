# Integrity check CLI

## Purpose

Offline audit that CI and agents run without starting MCP stdio. Catches package drift and playbook skips on feature branches; blocks merging mid-pipeline graph state to main.

## Execution Logic

1. `mindplan-mcp check [--base <ref>]` loads the territory graph, verifies every non-retired Foundation/Workflow has its prescribed package, rejects orphan `src/` package dirs, and enforces dirty-src ownership: uncommitted `src/` needs `in-progress` (or next in-progress / Bug fixing/in-review); committed diffs vs base allow review/shipped/cancelled/deprecated, but when `next.mdx` is open only next `in-progress`/`in-review` counts. Explicit `--base` fails closed on git errors.
2. `mindplan-mcp check --for-main` runs graph + package checks and fails if any Foundation/Workflow is `in-progress`/`in-review` (including `next`), or any Bug is `fixing`/`in-review`. Skips dirty-src.
3. Wired from `f-mcp-runtime` CLI (`node dist/index.js check`); this repo’s CI builds from source.

Package: `src/workflows/wf-integrity-check/`

- `check.ts` — `runIntegrityCheck`, dirty path collection, package/orphan/`--for-main` validators

## Acceptance Criteria

- Default check fails on orphan packages; uncommitted dirty src without in-progress fails; committed dirty while in-review/stable passes
- `--for-main` fails on mid-pipeline states and passes on draft/ready/stable/cancelled/deprecated graphs
- Explicit bad `--base` fails closed
- Exit code 0/1 with `Blocked:` lines for CI
