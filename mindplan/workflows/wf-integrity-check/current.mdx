# Integrity check CLI

## Purpose

Offline audit that CI and agents run without starting MCP stdio. On projects with prescribed packages, catches package drift and playbook skips on feature branches. On layout-free projects (`implementation_packages: "off"`), skips package and dirty-src ownership so brownfield `src/` trees are not forced into MindPlan layout. Always blocks merging mid-pipeline graph state to main via `--for-main`.

## Execution Logic

1. Load `mindplan/config.json` (missing → `implementation_packages: "required"`).
2. `mindplan-mcp check [--base <ref>]` loads the territory graph.
   - When `required`: verify every non-retired Foundation/Workflow has its prescribed package, reject orphan `src/` package dirs, and enforce dirty-src ownership: uncommitted `src/` needs `in-progress` (or next in-progress / Bug fixing/in-review); committed diffs vs base allow review/shipped/cancelled/deprecated, but when `next.mdx` is open only next `in-progress`/`in-review` counts. Explicit `--base` fails closed on git errors.
   - When `off`: skip package presence, orphan, and dirty-src ownership checks. Still validate the graph loads.
3. `mindplan-mcp check --for-main` runs graph (+ package checks only when `required`) and fails if any Foundation/Workflow is `in-progress`/`in-review` (including `next`), or any Bug is `fixing`/`in-review`. Skips dirty-src always.
4. Wired from `f-mcp-runtime` CLI (`node dist/index.js check`); this repo’s CI builds from source and stays on `required`.

Package: `src/workflows/wf-integrity-check/`

- `check.ts` — `runIntegrityCheck`, config-aware package/orphan/dirty-src/`--for-main` validators

## Acceptance Criteria

- Default check with `required` (or missing config): fails on orphan packages; uncommitted dirty src without in-progress fails; committed dirty while in-review/stable passes
- With `implementation_packages: "off"`: missing packages and dirty unowned `src/` do not fail; `--for-main` mid-pipeline ban still fails
- `--for-main` fails on mid-pipeline states and passes on draft/ready/stable/cancelled/deprecated graphs
- Explicit bad `--base` fails closed
- Exit code 0/1 with `Blocked:` lines for CI
