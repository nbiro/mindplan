# Steer the plan

## Purpose

Mutate the plan — human UI to advance status, link/unlink edges, open/discard `next`, and related compiler-gated edits. Same rules as MCP; never a bypass. Surfaces `Blocked:` clearly.

## Execution Logic

Package: `src/workflows/wf-console-mutate/`

- Actions via `f-console-bridge`: `update_node_status`, `link_nodes`, `unlink_nodes`, `open_next`, `discard_next`, optional `patch_node_territory`
- `force_unship` only after explicit in-UI confirmation and correct confirm token (playbook-aligned)
- On failure, show structured `Blocked:` reason; do not retry blindly
- On success, refresh shell see surfaces; show `changed_files`
- Territory context from `wf-territory-viewer` for the focused node

Belongs to: `j-plan-console`. Depends on: `wf-console-shell`, `wf-territory-viewer`, `f-console-bridge`, `f-design-system`.

## Acceptance Criteria

- Legal status transitions succeed and update the board/graph after refresh
- Illegal transitions show `Blocked:` with the compiler reason
- `force_unship` requires explicit human confirmation in the UI
- No direct filesystem rewrite of server-owned frontmatter fields
