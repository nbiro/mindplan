# Init a consumer project

## Purpose

Scaffold a consumer repo for MindPlan: empty `mindplan/` territory layout, always-on agent playbook, define-entities and plan-project skills, MCP example config, per-agent integration guides, root `AGENTS.md`, and `.cursorignore` so agents read territory via MCP.

## Execution Logic

Package: `src/workflows/wf-project-init/`

- `init.ts` — `initProject` plus install helpers:
  - `installAgentPlaybook` — `templates/agent/playbook.md` → `mindplan/agent/`
  - `installDefineEntitiesSkill` — skill under `mindplan/agent/skills/define-entities/`
  - `installPlanProjectSkill` — skill under `mindplan/agent/skills/plan-project/`
  - `installMcpExample` — `mcp.json.example`
  - `installAgentIntegrations` — guides into `mindplan/agent/integrations/`
  - `installRootAgentsMd` — root `AGENTS.md` from template
  - `installCursorIgnore` — blocks direct reads of territory MDX/map
- Invoked by CLI `mindplan-mcp init` in `f-mcp-runtime` (package root resolved by walking up to `templates/agent`)

Depends on: `f-territory-store`. Belongs to: `j-agent-onboarding`.

## Acceptance Criteria

- `mindplan-mcp init` creates/updates consumer mindplan + agent assets idempotently where intended
- Playbook, define-entities skill, plan-project skill, integrations, MCP example, AGENTS.md, and `.cursorignore` are installed from `templates/`
- Init helpers live in this package (not in the territory store)
- Smoke test covers init template install including plan-project
