# Init a consumer project

## Purpose

Scaffold a consumer repo for MindPlan: empty `mindplan/` territory layout, always-on agent playbook, define-entities skill, MCP example config, per-agent integration guides, root `AGENTS.md`, and `.cursorignore` so agents read territory via MCP.

## Execution Logic

Package: `src/workflows/wf-project-init/`

- `init.ts` — `initProject` plus install helpers:
  - `installAgentPlaybook` — `templates/agent/playbook.md` → `mindplan/agent/`
  - `installDefineEntitiesSkill` — skill under `mindplan/agent/skills/`
  - `installMcpExample` — `mcp.json.example`
  - `installAgentIntegrations` — guides into `mindplan/agent/integrations/`
  - `installRootAgentsMd` — root `AGENTS.md` from template
  - `installCursorIgnore` — blocks direct reads of territory MDX/map
- Invoked by CLI `mindplan-mcp init` in `f-mcp-runtime` (package root resolved by walking up to `templates/agent`)

Depends on: `f-territory-store`. Belongs to: `j-agent-onboarding`.

## Acceptance Criteria

- [x] `mindplan-mcp init` creates/updates consumer mindplan + agent assets idempotently where intended
- [x] Playbook, define-entities skill, integrations, MCP example, AGENTS.md, and `.cursorignore` are installed from `templates/`
- [x] Init helpers live in this package (not in the territory store)
- [x] Smoke test covers init template install
