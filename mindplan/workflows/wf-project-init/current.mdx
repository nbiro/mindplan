# Init a consumer project

## Purpose

Scaffold a consumer repo for MindPlan: empty `mindplan/` territory layout, project config (`mindplan/config.json`) choosing prescribed vs layout-free packages, always-on agent playbook, define-entities / plan-project / review-work skills, MCP example config, per-agent integration guides, root `AGENTS.md`, Cursor-native copies under `.cursor/skills/` and `.cursor/rules/mindplan.mdc`, `.cursorignore` that ignores the derived map (and agent asset copies under `mindplan/agent/**`) while allowing agent file tools to edit territory MDX prose, and `.cursor/permissions.json` that allowlists MindPlan MCP tools so Cursor Auto-review does not prompt on playbook-required graph mutations.

## Execution Logic

Package: `src/workflows/wf-project-init/`

- `init.ts` — `initProject` plus install helpers (exported for CLI wiring in `f-mcp-runtime`):
  - `installProjectConfig` — writes `mindplan/config.json` with `implementation_packages`: `required` (default / `--layout prescribed`) or `off` (`--layout free`). Missing config is treated as `required` by loaders elsewhere. Idempotent: create when missing; update only when an explicit `--layout` flag is passed
  - `installAgentPlaybook` — `templates/agent/playbook.md` → `mindplan/agent/`
  - `installDefineEntitiesSkill` — skill under `mindplan/agent/skills/define-entities/`
  - `installPlanProjectSkill` — skill under `mindplan/agent/skills/plan-project/`
  - `installReviewWorkSkill` — skill under `mindplan/agent/skills/review-work/`
  - `installCursorSkills` — copy the three skills into Cursor discovery paths from package templates (idempotent, skip if dest exists):
    - → `.cursor/skills/mindplan-define-entities/`
    - → `.cursor/skills/mindplan-plan-project/`
    - → `.cursor/skills/mindplan-review-work/`
    Returns one install result per skill path (or a single aggregate — implementer’s choice) so the CLI can report each path. Source is `templates/agent/skills/…`, not the on-disk `mindplan/agent` copy.
  - `installCursorRule` — write `.cursor/rules/mindplan.mdc` when missing: Cursor rule frontmatter (`alwaysApply: true`) plus the playbook body from `templates/agent/playbook.md`
  - `installMcpExample` — `mcp.json.example`
  - `installAgentIntegrations` — guides into `mindplan/agent/integrations/` (Cursor guide documents that init installs `.cursor/skills` + `.cursor/rules`; manual copy only if those files were deleted)
  - `installRootAgentsMd` — root `AGENTS.md` from template
  - `installCursorIgnore` — installs `templates/agent/cursorignore` (ignores `mindplan/map.md` + `mindplan/agent/**`; does **not** block `current.mdx` / `next.mdx`). Comment must match real behavior: init copies agent instructions to `.cursor/rules` and `.cursor/skills`
  - `installCursorPermissions` — installs `templates/agent/permissions.json` → `.cursor/permissions.json` when missing

CLI call/report of these helpers lives in `f-mcp-runtime` (paired `next`). Smoke asserts for Cursor paths live in `wf-test-harness` (paired `next`).

Depends on: `f-territory-store`. Belongs to: `j-agent-onboarding`. Consumer use case: `wf-layout-free`. Fixes: `bug-init-cursor-skills` (with paired `f-mcp-runtime` + `wf-test-harness` nexts).

## Acceptance Criteria

- `mindplan-mcp init` creates/updates consumer mindplan + agent assets idempotently where intended
- `init` writes `mindplan/config.json`; `--layout free` sets `implementation_packages` to `off`; default/prescribed sets `required`
- Playbook, define-entities, plan-project, and review-work skills, integrations, MCP example, AGENTS.md, `.cursorignore`, and `.cursor/permissions.json` are installed from `templates/`
- This package exports `installCursorSkills` and `installCursorRule` that install `.cursor/skills/mindplan-define-entities|plan-project|review-work/` (each with `SKILL.md`) and `.cursor/rules/mindplan.mdc` (alwaysApply frontmatter + playbook body), idempotently
- Template `.cursorignore` does not list `mindplan/**/current.mdx` or `mindplan/**/next.mdx`, and its comment accurately describes the `.cursor/rules` + `.cursor/skills` copies
- Cursor integration guide states that init installs those Cursor paths (manual copy only if missing)
- Template `.cursor/permissions.json` allowlists `mindplan:*` and steers Auto-review for playbook mutations
- Init helpers live in this package (not in the territory store)
- CLI wiring/reporting is owned by `f-mcp-runtime` next; Cursor-path smoke asserts by `wf-test-harness` next
