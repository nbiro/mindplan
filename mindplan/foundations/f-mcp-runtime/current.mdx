# MCP stdio runtime

## Purpose

Assembler — the `mindplan-mcp` process: MCP stdio server and CLI shell that registers tools owned by Workflows and boots from thin `src/index.ts`. Resolves the package root (for `templates/`) by walking up to `templates/agent`.

## Shared Substrate Spec

Package: `src/foundations/f-mcp-runtime/`

- `server.ts` — MCP server registration; CLI (`init`, `view`, help); wires compiler + store + search + view projection; registers read tools (`wf-query-graph`), write tools (`wf-mutate-graph`), and `export_mindplan_view` (`wf-export-views`); auto-persists `mindplan/map.md` after successful graph mutations

Depends on: `f-compiler-rules`, `f-territory-store` (also uses graph-search and view-projection at runtime).

## Acceptance Criteria

- [x] MCP tools listed in Workflow ownership modules are registered and callable over stdio
- [x] CLI `init` / `view` / help work against the resolved package root
- [x] Mutations that succeed refresh the Mermaid map on disk
- [x] Verified via `npm test` (smoke)
