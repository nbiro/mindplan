# MCP stdio runtime

## Purpose

Assembler — the `mindplan-mcp` process: MCP stdio server and CLI shell that registers tools owned by Workflows and boots from thin `src/index.ts`. Resolves the package root (for `templates/`) by walking up to `templates/agent`. Mutate handlers return `changed_files` so agents and humans can see which paths MCP wrote. CLI includes `init`, `view`/`export`, and offline `check` (integrity). Registers `force_unship` for mistaken-ship recovery (user-confirmed). CLI `init` installs consumer agent assets including `.cursor/permissions.json`.

## Shared Substrate Spec

Package: `src/foundations/f-mcp-runtime/`

- `server.ts` — MCP server registration; CLI (`init`, `view`, `check`, help); wires compiler + store + search + view projection + `wf-integrity-check`; registers read/write tools; auto-persists `mindplan/map.md` after successful graph mutations; `update_node_status` documents `cancelled`

Depends on: `f-compiler-rules`, `f-territory-store` (also uses graph-search, view-projection, and `wf-integrity-check` at runtime).

## Acceptance Criteria

- MCP tools listed in Workflow ownership modules are registered and callable over stdio
- CLI `init` / `view` / `check` / help work against the resolved package root
- `check` and `check --for-main` exit 0/1 with `Blocked:` lines
- Mutations that succeed refresh the Mermaid map on disk
- Verified via `npm test`
