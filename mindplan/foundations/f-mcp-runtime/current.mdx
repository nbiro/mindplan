# MCP stdio runtime

## Purpose

Assembler — the `mindplan-mcp` process: MCP stdio server and CLI shell that registers tools owned by Workflows and boots from thin `src/index.ts`. Resolves the package root (for `templates/`) by walking up to `templates/agent`. Mutate handlers return `changed_files` so agents and humans can see which paths MCP wrote. CLI includes `init`, `view`/`export`, and offline `check` (integrity). Registers `force_unship` for mistaken-ship recovery (user-confirmed). CLI `init` installs consumer agent assets including define-entities, plan-project, review-work, and thin code-review skills under `mindplan/agent/`, Cursor-native copies under `.cursor/skills/` (including `mindplan-code-review`) and `.cursor/rules/mindplan.mdc`, and `.cursor/permissions.json`. Supports `init -f` / `--force` so package upgrades can reinstall those assets over existing copies.

## Shared Substrate Spec

Package: `src/foundations/f-mcp-runtime/`

- `server.ts` — MCP server registration; CLI (`init`, `view`, `check`, help); wires compiler + store + search + view projection + `wf-integrity-check`; registers read/write tools; auto-persists `mindplan/map.md` after successful graph mutations; `update_node_status` documents `cancelled`; `init` calls install helpers from `wf-project-init`, including:
  - playbook, define-entities / plan-project / review-work / code-review (mindplan/agent), MCP example, integrations, AGENTS.md, `.cursorignore`, `.cursor/permissions.json`
  - `installCursorSkills`, `installCursorRule` — report each Cursor skill path and `mindplan.mdc` like other install results
  - Init argv parser accepts `--layout free|prescribed`, `-f`, and `--force`. `-f` / `--force` set `force: true` on all agent-asset installers. `--layout` still independently forces config rewrite. Help text documents both.
  - Note: `view`/`export` already uses `-f` as shorthand for `--format`; that is a different command — init's `-f` means force overwrite only under `init`.

Depends on: `f-compiler-rules`, `f-territory-store` (also uses graph-search, view-projection, and `wf-integrity-check` at runtime). Helper implementations live in `wf-project-init` (paired `next`); this Foundation only imports, calls, and reports them. Smoke coverage for force overwrite: `wf-test-harness` (paired `next`).

## Acceptance Criteria

- MCP tools listed in Workflow ownership modules are registered and callable over stdio
- CLI `init` / `view` / `check` / help work against the resolved package root
- CLI `init` installs and reports `review-work` and thin `code-review` skills (mindplan/agent + Cursor `mindplan-code-review`)
- CLI `init` calls `installCursorSkills` and `installCursorRule` and reports their install paths (`.cursor/skills/mindplan-*`, `.cursor/rules/mindplan.mdc`)
- CLI `init -f` and `init --force` pass `{ force: true }` to every agent-asset installer; help documents `-f` / `--force`
- Bare `init` (no force) remains skip-if-exists; `--layout` still overwrites config independently of `-f`
- `check` and `check --for-main` exit 0/1 with `Blocked:` lines
- Mutations that succeed refresh the Mermaid map on disk
- Verified via `npm test` (force-overwrite asserts owned by `wf-test-harness` next)
