# MCP stdio runtime

## Purpose

Assembler â€” the `mindplan-mcp` process: MCP stdio server and CLI shell that registers tools owned by Workflows and boots from thin `src/index.ts`. Resolves the package root (for `templates/`) by walking up to `templates/agent`. Mutate handlers return `changed_files` so agents and humans can see which paths MCP wrote. CLI includes `init`, `view`/`export`, and offline `check` (integrity). Registers `force_unship` for mistaken-ship recovery (user-confirmed). CLI `init` installs consumer agent assets including define-entities, plan-project, review-work, and thin code-review skills under `mindplan/agent/`, Cursor-native copies under `.cursor/skills/` (including `mindplan-code-review`) and `.cursor/rules/mindplan.mdc`, and `.cursor/permissions.json`.
