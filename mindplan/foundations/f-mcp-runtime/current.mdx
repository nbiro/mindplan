# MCP stdio runtime

## Purpose

Assembler — the `mindplan-mcp` process: MCP stdio server and CLI shell that registers tools owned by Workflows and boots from thin `src/index.ts`. Resolves the package root (for `templates/`) by walking up to `templates/agent`. Mutate handlers return `changed_files` so agents and humans can see which paths MCP wrote. Registers `force_unship` for mistaken-ship recovery (user-confirmed). CLI `init` installs consumer agent assets including `.cursor/permissions.json`.

## Shared Substrate Spec

Package: `src/foundations/f-mcp-runtime/`

- `server.ts` — MCP server registration; CLI (`init`, `view`, help); wires compiler + store + search + view projection; registers read tools (`wf-query-graph`), write tools (`wf-mutate-graph` including `force_unship`), and `export_mindplan_view` (`wf-export-views`); auto-persists `mindplan/map.md` after successful graph mutations (including `force_unship`); graph mutate tool results include `changed_files: string[]` (and `path` on territory patches); `init` calls `wf-project-init` install helpers including `installCursorPermissions`

Depends on: `f-compiler-rules`, `f-territory-store` (also uses graph-search and view-projection at runtime).

## Acceptance Criteria

- MCP tools listed in Workflow ownership modules are registered and callable over stdio
- CLI `init` / `view` / help work against the resolved package root
- CLI `init` installs `.cursor/permissions.json` when missing (via `installCursorPermissions`)
- Mutations that succeed refresh the Mermaid map on disk
- Graph mutate tools return `changed_files` for paths written or deleted
- `force_unship` clears `shipped_at` via `patchFrontmatter(..., { clearShippedAt: true })` after Rule 10 validation
- Verified via `npm test` (smoke)
