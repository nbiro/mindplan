# Territory filesystem store

## Purpose

Infra — GitOps persistence for the MindPlan territory under `mindplan/`. Discovers the graph from MDX frontmatter, scaffolds entities (and prescribed `src/` packages when configured), and owns `next.mdx` open/promote/discard plus territory body patches. Supports clearing `shipped_at` when recovering a mistaken ship via `force_unship`. Reads `mindplan/config.json` so layout-free projects can opt out of prescribed implementation packages.

## Shared Substrate Spec

Package: `src/foundations/f-territory-store/`

- `store.ts` — path helpers (`mindplanRoot`, `entityDir`, `implementationDir`, …); frontmatter parse/write; `discoverNodes` / `discoverEdges` / `loadGraph`; scaffold entity + optional implementation package; `readMarkdown` / `writeMarkdown` / `patchFrontmatter` (optional `{ clearShippedAt: true }` removes the `shipped_at:` line) / edge writers (respecting next-slot edge writes); `openNextSlot` / `discardNextSlot` / `promoteNextSlot`; `patchNodeTerritory` (slot resolution `current`|`next`); `countUncheckedBoxes`; `getNodeImplementation`; **project config loader** for `implementation_packages` (`required` | `off`, default `required` when file missing)
- When `implementation_packages` is `off`: `scaffoldImplementationPackage` is a no-op; `getNodeImplementation` reports packages off / no implied prescribed root (callers must not treat absence as integrity failure)
- When `required`: Foundations/Workflows get prescribed packages under `src/foundations/<id>/` or `src/workflows/<id>/` (unchanged)

Consumer init/install helpers live in `src/workflows/wf-project-init/init.ts` (not this Foundation).

Depends on: `f-domain-model`.

## Acceptance Criteria

- Graph load matches on-disk `current.mdx` (+ optional `next.mdx`) frontmatter
- `open_next` seeds body and outgoing edges; `ship` promotes next over current; `discard_next` leaves current untouched
- Territory patches can target `current` or `next`; default to `next` when an evolution is open on that node
- With `required` (or missing config): Foundations/Workflows get prescribed packages under `src/foundations/<id>/` or `src/workflows/<id>/`
- With `off`: `create_node` / scaffold creates territory only; no prescribed package dirs; `get_node_implementation` surfaces packages-off clearly
- `patchFrontmatter(..., { clearShippedAt: true })` removes `shipped_at` from `current.mdx`
- Verified via `npm test`
