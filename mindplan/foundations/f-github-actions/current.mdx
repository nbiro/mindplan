# GitHub Actions assembler

## Purpose

Assembler that mounts CI and release jobs for this repository on GitHub Actions. Owns the **CI substrate** and **OIDC permission conventions** for any job that publishes. Does **not** own the release workflow YAML body â€” that is `wf-npm-tag-publish`.

## Ownership split

| Surface | Owner |
|---------|--------|
| `.github/workflows/ci.yml` (test CI) | **This Foundation** |
| OIDC / `registry-url` / `id-token: write` conventions for publish jobs | **This Foundation** (applies `f-npm-registry` contract) |
| `.github/workflows/publish.yml` (tag-driven release job steps) | **`wf-npm-tag-publish` only** |

Do not duplicate Atomic Ops for writing `publish.yml` on this Foundation.

## Shared Substrate Spec

### Existing CI

- File: `.github/workflows/ci.yml`
- Triggers: push/PR to `main` / `master`
- Matrix: Node 18, 20, 22 â€” `npm ci` â†’ `npm run build` â†’ `npm test`
- Release automation MUST NOT weaken this gate

### Conventions for publish jobs (consumed by `wf-npm-tag-publish`)

When `wf-npm-tag-publish` authors `.github/workflows/publish.yml`, it MUST follow:

- Permissions: `contents: read`, `id-token: write` (per `f-npm-registry`)
- `actions/setup-node` with `registry-url: https://registry.npmjs.org`
- Node: pin one LTS aligned with CI (prefer 22 or 20); no matrix publish
- Invoke the shared publish script from `wf-npm-publish` in CI mode (that mode **must** run `npm test` â€” do not drop tests for a bare `npm publish`)

### Depends on

- `f-npm-registry` â€” OIDC / Trusted Publishing **contract** (docs); live npm binding is still `wf-npm-tag-publish`

### Out of scope

- Authoring or owning `publish.yml` job steps (Workflow `wf-npm-tag-publish`)
- Publish script contents (Workflow `wf-npm-publish`)
- Changelog automation

## Implementation

Package: `src/foundations/f-github-actions/`. CI YAML lives under `.github/workflows/ci.yml` as the deployable surface this Assembler owns. Release YAML is written by `wf-npm-tag-publish` under `.github/workflows/publish.yml` (repo-root workflow path is an intentional exception to prescribed packages, same as CI).
