# Domain model

## Purpose

Infra — shared TypeScript taxonomy for MindPlan nodes, edges, pipeline states, and the optional `next` evolution slot. Every other Foundation and Workflow imports these types; there is no I/O here.

## Shared Substrate Spec

Package: `src/foundations/f-domain-model/`

- `types.ts` — `NODE_TYPES`, execution/production/Bug/Journey state machines and legal transitions (including terminal `cancelled` for pre-ship abandon), edge types (`belongs_to` / `depends_on` / `affects`), `MindPlanNode` / `MindPlanEdge` / `MindPlanGraph`, next-slot pipeline states, `GRAPH_VERSION`, helpers (`isExecutionState`, `isOpenBugState`, …)

Depends on: none.

## Acceptance Criteria

- All NodeTypes, edge types, and state machines used by the compiler and store are defined here
- Next-slot types support stable-id evolution without successor node ids
- `cancelled` is a terminal execution state reachable from draft/ready/in-progress/in-review
- `GRAPH_VERSION` is reported by `get_mindplan_graph`
- Verified via `npm test`
