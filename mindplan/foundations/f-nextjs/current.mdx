# Next.js assembler

## Purpose

Assembler — Next.js App Router runtime that mounts console Workflow packages into a local deployable UI. Owns app bootstrap, routing shell hooks, and how Workflow UIs are composed — not stakeholder use-case behaviour (board, graph, mutate, model).

## Shared Substrate Spec

Package: `src/foundations/f-nextjs/`

- App Router entry and layout primitives for the MindPlan console
- Mount points for Workflow packages under `src/workflows/wf-*`
- Local-dev conventions (project root env / launch) consumed by `wf-console-shell`

MUST NOT own status-board, graph-explore, territory, mutate, or model screens — those are Workflows.

## Acceptance Criteria

- Console Workflows can be mounted as App Router routes/packages
- Assembler stays free of product use-case logic
- Local `next dev` (or equivalent) can boot the shell against a chosen project root
