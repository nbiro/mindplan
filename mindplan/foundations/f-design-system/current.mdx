# Console design system

## Purpose

Design system — tokens, typography, and dumb presentational primitives for the MindPlan console. Shared look-and-feel for shell, board, graph chrome, territory, and forms. No product behaviour or MindPlan compiler calls.

## Shared Substrate Spec

Package: `src/foundations/f-design-system/`

- CSS variables / theme tokens (type, color, space) — avoid default AI-slop palettes; pick a deliberate console identity
- Dumb primitives: layout, text, buttons, inputs, badges (including state badge chrome used by viewers)
- No data fetching, no bridge calls, no graph layout logic

Consumed by console Workflows and by `f-xyflow` for node/edge styling hooks.

## Acceptance Criteria

- Workflows can compose screens without inventing one-off token stacks
- Components are presentational only (props in, markup out)
- State badge / type colors are consistent across board and graph
