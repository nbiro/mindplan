# Compiler rules engine

## Purpose

Infra — the MindPlan compiler. Every graph mutation that can change state or edges is validated here; violations throw `Blocked: …` and must not be retried blindly. Includes Rule 10 force-unship recovery and pre-ship `cancelled` abandon gates.

## Shared Substrate Spec

Package: `src/foundations/f-compiler-rules/`

- `rules.ts` — find-node helpers; Rule 1 Ghost Workflow / Ghost Bug; Rule 2 Infrastructure First; Rule 3 Completion Check; taxonomy / dependency closure; status transition tables (including `cancelled`); Journey state recompute; production stability from open Bugs; `open_next` / `discard_next` / ship-promote gates (Rule 9); `resolveForceUnship` (Rule 10); cancel gates (no open `next`, no active dependents); shipped Workflow title/description freeze on `current`

Depends on: `f-domain-model`. Uses `countUncheckedBoxes` from `f-territory-store`.

## Acceptance Criteria

- Illegal transitions and edge shapes are rejected with `Blocked:` messages
- `cancelled` is allowed from draft/ready/in-progress/in-review; blocked with active dependents or open `next`
- Journey and `stable`/`unstable` are computed, never set manually through this layer’s callers
- Ship and `in-review` require Completion Check against the active territory file (`current` or `next`)
- Verified via `npm test`
