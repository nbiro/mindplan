# Compiler rules engine

## Purpose

Infra — the MindPlan compiler. Every graph mutation that can change state or edges is validated here; violations throw `Blocked: …` and must not be retried blindly. Includes Rule 10 force-unship recovery for mistaken ships.

## Shared Substrate Spec

Package: `src/foundations/f-compiler-rules/`

- `rules.ts` — find-node helpers; Rule 1 Ghost Workflow / Ghost Bug; Rule 2 Infrastructure First; Rule 3 Completion Check (unchecked Atomic Ops); taxonomy / dependency closure; status transition tables; Journey state recompute; production stability from open Bugs; `open_next` / `discard_next` / ship-promote gates (Rule 9 Next Evolution); `resolveForceUnship` / `forceUnshipConfirmToken` (Rule 10 Force Unship); shipped Workflow title/description freeze on `current`

Depends on: `f-domain-model`. Uses `countUncheckedBoxes` from `f-territory-store`.

## Acceptance Criteria

- Illegal transitions and edge shapes are rejected with `Blocked:` messages
- Journey and `stable`/`unstable` are computed, never set manually through this layer’s callers
- Ship and `in-review` require Completion Check against the active territory file (`current` or `next`)
- `resolveForceUnship` enforces confirm token, no-next, no shipped dependents, and pre-ship targets only
- Illegal production retreats via `update_node_status` mention `force_unship`
- Verified via `npm test`
